# AGENTS.md

## 基本方針

このリポジトリでは、仕様書に基づく最小実装を優先する。

Codexは、依頼された作業範囲を超えて、機能追加・設計変更・大規模リファクタリングを行ってはいけない。

公開I/Fは `python scripts/<script>.py ...` の直接実行に固定する。

標準フローでは以下を原則とする。

- 1 stage = 1 script
- 1 script = 1 主成果物
- stage script は主成果物を作る責務に絞る
- stage script は出力前の自己検査を内部で完結させる
- `pipeline.py` のような中継層は作らない
- 横断 validator は持たない
- validator や補助処理は標準フロー外として扱う

## スコープ管理

以下は禁止する。

- 仕様書にない機能追加
- 将来拡張のための抽象化
- 不要な設定項目の追加
- 互換層・アダプタ・ラッパーの過剰な追加
- 大規模なディレクトリ構成変更
- 依頼と直接関係しないリファクタリング
- 見つけた問題のついで修正
- TODOの勝手な実装

問題点を見つけた場合は、実装せず `NOTES.md` に記録する。

## 仕様変更が必要そうな場合

仕様を変えた方が自然・簡潔・安全だと判断した場合は、実装を進めずに停止する。

その場合は、以下を提示する。

- 現行仕様の問題点
- 仕様を守る場合の実装案
- 仕様を変更する場合の実装案
- 変更した方がよい理由
- 影響範囲

承認なしに仕様変更してはいけない。
承認なしに複雑な回避実装を追加してはいけない。

## テスト

コードを変更した場合は、作業完了前に以下を実行する。

```bash
.venv/bin/python -m pytest -q
```

テストを実行できない場合は、理由と未確認範囲を完了報告に記載する。
ドキュメントのみの変更では、pytest の実行を必須としない。

## 作業完了時の報告

完了時は以下を簡潔に報告する。

- 変更した内容
- 変更したファイル
- 実行した確認コマンド
- 未確認のこと
- 仕様上の懸念
